## P2 · 应届生求职的真问题

每年 **1158 万**应届生（2024 教育部数据）涌入就业市场。**平均每人投出 60+ 份简历**（牛客网年度调研），**70% 没有回音**。

但真正让应届生焦虑的，不是投递数量——是**三个看不见**：

### 看不见 1：HR 的真实评价
你投出去的简历，HR 用 1 分钟扫过，淘汰理由是什么？是学校？是项目深度？是 keyword 不匹配？没人告诉你。下一次怎么改？只能猜。

### 看不见 2：公司的隐性门槛
JD 写"本科及以上"，实际录取者 92% 是 211 起步。JD 写"不限性别"，实际同岗位男女比是 7:3。JD 写"35 岁以下"是违法，但实际筛简历时年龄是第一个被看的字段。**这些 JD 不写但人人都用的隐性维度**，应届生进不了"局内人视角"。

### 看不见 3：决策的后悔成本
拿到字节 + 美团两个 offer，选哪个？三年后你会怎样？你的同学选了另一个，他三年后怎样？没人能告诉你**反事实**。决策只能赌一次，且永远没法验证。

**现有 AI 求职工具解决的是"投递效率"这种局部问题**——简历优化、面试模拟、JD 匹配。**但应届生真正需要的是上面三个"看不见"的解药**。

## P3 · 解决方案：把整个春招"虚拟跑一遍"

### 用户体验流程（4 步）

```
[1] 上传简历 + GitHub + Blog
        ↓
[2] AI 用你的资料建立 RAG persona（"认识你"）
        ↓
[3] 数字分身进入沙盘宇宙：
    - 49 家虚构公司（按行业聚成"星团"，每家有独立 HR Agent）
    - 200 个竞争者池（你的真实竞争对手画像）
    - 模拟 13 周完整春招
    - 并行跑 1000 次（蒙特卡洛）
        ↓
[4] 输出：
    - 1000 次平行宇宙的 KPI 统计
    - 各公司 offer 概率分布
    - 决策树（每次 sim 的关键岔路）
    - 反事实分析（"如果你改 X，结果如何变化"）
    - "采访 HR"：评委/用户可现场提问任意公司的 HR Agent
```

### 核心输出：从"建议"到"统计预演"

不是说"你应该投这几家"，是说：

> **"在 1000 次平行宇宙里，你最可能的去向是焰火（40% 概率），中位薪资 49 万。**
> **如果你把项目含金量再提升 15 分（相当于多做一个有深度的项目），**
> **平均 offer 数从 3.7 涨到 4.5，平均薪资 +14 万。"**

这是**统计意义上的"后悔药"**——不是教你怎么做，是让你**看到不做的代价**。

## P5 · 核心创新 1：结构反转

### 现状：单向投递

```
求职者 → 投简历 → HR 看 1 分钟 → 90% 没回音
```

零和、不透明、不可逆。

### 我们：双边沙盘 + 1000 次预演

```
求职者数字分身  ⇌  公司 HR Agent
        ↑              ↑
    1000 次平行 sim   独立性格 + 隐性门槛
        ↓
   反事实 + 决策树 + 统计报告
```

不是优化"投递"这个动作，是**重构"投递→筛选"的市场结构**——把一次性的赌博，变成可重复的统计预演。

## P6 · 核心创新 2 + 3：载体反转 + 建议→预演

### 载体反转：从 PDF 到可对话化身

| 传统 | 我们 |
|---|---|
| 简历 = 静态 PDF，发出去就石沉大海 | 数字分身 = 可被采访的 AI 化身 |
| HR 看 1 分钟，看不深 | HR Agent 可与化身"展开 N 轮对话" |
| 候选人对 HR 不透明 | 化身可"现场被采访"，评委/用户问任意问题 |

### 建议→预演

| 传统 AI 求职 | 我们 |
|---|---|
| "你简历项目部分写得不够清晰，建议加 STAR 法则" | **"如果你把项目含金量从 65 提到 80，1000 次 sim 里 offer 数从 3.7 → 5.2，薪资 +14 万。如果继续提到 90 收益递减。"** |
| 建议是模糊的"你应该做 X" | 预演是"做 X 后 N 次模拟的统计结果" |
| 没有可量化的反馈 | 滑动条调三个维度，实时看影响 |

这是从"知识传授"到"决策科学化"的跃迁。

## P7 · 技术架构

### 系统架构图

```
[用户浏览器]
       ↓
   nginx (Azure VM)
       ├─ /               → Vue3 SPA (Three.js 3D 沙盘 + D3.js 数据可视化)
       └─ /api/*          → FastAPI (uvicorn :8000)
                              ↓
        ┌─────────────────────┼─────────────────────┐
        ↓                     ↓                     ↓
   SimulationEngine     RAG-based persona      Admin CRUD
        ↓                                          ↓
   ┌────┴─────┬───────────┐                    JSON 存储
   ↓          ↓           ↓                  + 文件锁
CandidateAgent CompanyHRAgent InterviewerAgent
   ↓          ↓           ↓
   └──────────┴───────────┘
              ↓
       LLMRouter（tier 路由）
              ↓
   ┌──────────┼──────────┬──────────┐
   ↓          ↓          ↓          ↓
DeepSeek    Qwen       Kimi       GLM ...
（PRIMARY） （配角）   （备用）    （扩展）
```

### 关键技术点

| 模块 | 技术选型 | 设计取舍 |
|---|---|---|
| **LLM 抽象层** | tier 路由（PRIMARY/SECONDARY/BACKGROUND）+ OpenAI 兼容协议 | 切换模型 = 改 .env 一行，业务代码 0 行变动。覆盖大陆主流模型 |
| **Multi-Agent** | 5 类 Agent 各司其职：求职者分身 / 公司 HR / 面试官 / 竞争者池（规则模拟）/ 市场环境 | 不用 LangGraph（过度抽象），asyncio + 自研编排更轻 |
| **反事实分析** | baseline + N mutation 变体，monotonic 单调插值 | 真跑 1000 次每个 mutation 不现实（成本爆炸）。统计模型 + 真 sim 校准 |
| **持久化** | JSON 文件 + threading.Lock + 原子写（tempfile → rename） | 50 公司 + 200 persona 规模 JSON 全文 < 1MB，每次重读 50ms 可接受，不上 SQLite 减少复杂度 |
| **动态市场** | admin CRUD endpoints + 数据热更新 | 公司/求职者随时加入退出，sim 启动 snapshot 最新数据 |
| **合规** | sanitizer 双层（prompt 约束 + 落盘前扫描真名）+ 全 synthetic 数据 | 零 PII，全代号公司 |

### 我们刻意**不**做的事

| 不做 | 为什么 |
|---|---|
| LoRA 微调 | 动态市场场景下 LoRA 是反模式——训完就过期，无法跟随公司/求职者池变化 |
| 真实公司数据 | 合规 0 风险优先。BP 此处反复强调 |
| LangGraph 编排 | 13 周固定 tick 循环不需要"路径不确定的协作" |
| 声音克隆 / 区块链 / 元宇宙 buzzword | 与产品价值无关，评委一看就识破 |

### C 端：付费跑 sim

| 套餐 | 价格 | 包含 |
|---|---|---|
| 单次预演 | 9.9 元 | 1 次 1000-run sim + 完整报告 |

| 决策咨询 | 199 元/次 | 1 对 1 真人解读 + 个性化策略 |

**单位经济**：
- 单次 sim 成本（DeepSeek API）≈ 0.1 元（13 周 × ~30 LLM 调用 × 0.003 元/次）
- 单次售价 9.9 元 → **毛利 99%**
- 反事实 + 报告增量成本 ≈ 0（线性插值，不消耗 LLM）

### B 端：反向品牌报告

给企业 HR / 招聘官出"你公司在应届生眼里的真实标签"报告：

- 你的 hiring_bar 在市场上的相对位置
- 你的 culture_tags 给应届生留下什么印象
- 同行业竞争对手的隐性门槛对比
- 你的 JD 文本能吸引哪类候选人画像

**价格**：5-20 万/年（B 端 SaaS 订阅制），高度定制化。

### 平台分润

与招聘平台合作：

- 用户在我们这跑 sim → 拿到目标公司列表 → 跳转招聘平台投递
- 我们抽取每次成功投递的 5-10% 佣金
- **招聘平台得到**：用户停留时长 + 投递质量提升（用户更精准）
- **我们得到**：流量入口 + 用户增长

### 飞轮闭环

```
应届生用我们跑 sim → 调改简历策略 → 在招聘平台投出更精准简历
        ↑                                              ↓
        └──────── 平台用户粘性提升，反过来推广我们 ────┘
```

## P9 · 数据合规（评委关键关注点）

### 我们所有数据都是合成的

| 数据类型 | 来源 | 合规处理 |
|---|---|---|
| 49 家虚构公司画像 | LLM 生成 + 模板填充 | 代号化（A 厂 / AI-α / 金-2 等），不挂任何真公司名 |
| 200 个求职者 persona | LLM 生成 | 化名（"张同学"等），零真实信息 |
| HR Agent 行为 | 公开 JD 风格 + 公开面试题库泛化 | 不引用任何真实 HR 行为案例 |
| 用户上传材料 | 用户自己提供 | 仅 session 内存使用，**不入数据库**，关闭浏览器即丢 |

### sanitizer 双层兜底

```
Layer 1 - prompt 约束：
  "不准出现任何真实公司名（包括 inspired_by_hint）。
   用'某头部短视频平台量级'这种泛指。"

Layer 2 - 落盘前扫描：
  字典匹配 60+ 真公司名（字节/腾讯/阿里/华为/...）+ 顶尖院校名
  发现立即替换为代号
```

实际效果：5/49 家公司在 LLM 生成时漏写了真名，**100% 被 sanitizer 在落盘前替换**。

### 数据合规声明（提交材料独立 1 页）

我们承诺：

1. 用户上传的简历仅 session 内存使用，进程退出即销毁
2. 不收集 PII（个人身份信息）
3. 不向第三方共享任何数据
4. 沙盘里所有 49 公司 + 200 求职者全部为合成数据，零真人参与

## P13 · 风险与缓解

| 风险 | 缓解策略 |
|---|---|
| **数据合规**（评委 #1 关注） | synthetic 数据 + sanitizer 双层 + 用户材料仅会话内存（已实现） |
| **LLM 成本爆炸** | tier 路由（PRIMARY Claude/DeepSeek，群演 Qwen-Turbo），单次 sim < 0.1 元 |
| **sim 真实性质疑** | 复赛/决赛阶段接入公开真实公司数据建模 + 第三方业内人士背书 |
| **用户上传简历隐私** | 仅 session 内存，进程退出即销毁，不入库 |
| **同类竞品快速抄袭** | 工程深度（Multi-Agent + 反事实 + 动态市场）+ 持续迭代 + 早期抢占用户心智 |
| **招聘平台抵触合作** | 我们的价值是"导流 + 提质"，不抢平台核心业务（投递撮合），合作而非竞争 |
| **应届生付费意愿低** | 春招/秋招前后 4 周是付费高峰，其余时间 freemium 留存。决策咨询是高客单补充 |
