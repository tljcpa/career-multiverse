## P2 · 应届生求职的真问题

每年 **1158 万**应届生（2024 教育部数据）涌入就业市场。**平均每人投出 60+ 份简历**（牛客网年度调研），**70% 没有回音**。

但真正让应届生焦虑的，不是投递数量——是**三个看不见**：

### 看不见 1：HR 的真实评价
你投出去的简历，HR 用 1 分钟扫过，淘汰理由是什么？是学校？是项目深度？是 keyword 不匹配？没人告诉你。下一次怎么改？只能猜。

### 看不见 2：公司的隐性门槛
JD 写"本科及以上"，实际录取者 92% 是 211 起步。JD 写"不限性别"，实际同岗位男女比是 7:3。JD 写"35 岁以下"是违法，但实际筛简历时年龄是第一个被看的字段。**这些 JD 不写但人人都用的隐性维度**，应届生进不了"局内人视角"。

### 看不见 3：决策的后悔成本
拿到字节 + 美团两个 offer，选哪个？三年后你会怎样？你的同学选了另一个，他三年后怎样？没人能告诉你**反事实**。决策只能赌一次，且永远没法验证。

**现有 AI 求职工具解决的是"投递效率"这种局部问题**——简历优化、面试模拟、JD 匹配。**但应届生真正需要的是上面三个"看不见"的解药**。

### 往上看一层：这不只是个人的问题

天气可以预报、城市可以用数字孪生优化，但**人才市场至今只能靠经验决策**——个人凭感觉投递、高校凭经验做就业指导、企业凭直觉定招聘策略。整个市场缺一个"先实验、再决策"的地方。我们想做的，就是**第一个可实验的人才市场**：个人是入口，高校和企业是真正的受益方。

## P3 · 解决方案：把整个春招"虚拟跑一遍"

### 用户体验流程（4 步）

```
[1] 上传简历 + GitHub + Blog
        ↓
[2] AI 一次性多任务评估你的简历（抽取核心字段 + 5 维内部画像评分 + 每维评分理由 + 学校档判定 + 学历加成）
        ↓
[3] 数字分身进入沙盘宇宙：
    - 约 300 家虚构公司（按行业聚成"星团"，每家有独立 HR Agent）
    - 约 2000 人竞争者池（你的真实竞争对手画像）
    - 模拟 13 周完整春招
    - 真跑数十次（本轮 30 次）真实多智能体对局，再统计推断分布
        ↓
[4] 输出：
    - 数十次真实模拟推断出的 KPI 统计分布
    - 各公司 offer 概率分布
    - 决策树（每次 sim 的关键岔路）
    - 反事实分析（"如果你改 X，结果如何变化"）
    - "采访 HR"：可提问任意公司的 HR Agent（**支持多轮持续对话，会话级 chat history 持久化，不是一问一答**，演示视频中展示）
    - **AI 教练个性化建议**：基于本人多次 sim 统计 + 5 维评分 + 学校档/学历档，LLM 给出"top 强项"/"最大瓶颈"/"3 条可执行建议"

> **关于"平行宇宙"的真实采样架构（透明披露）**：每次评估会真实完整跑 **数十次 LLM Multi-Agent 对局**（本轮基线跑满 30 次，每次约 30 个 LLM 调用、5 类 agent × 多轮交互、每次跑满 13 周），再用统计聚合（bootstrap 重采样 + 蒙特卡洛扰动）把这数十次真跑推断成稳定的分布。"平行宇宙"是概念表述——我们真跑的是数十次，**不谎称"真跑 1000 次"**。前端进度条文案 + 产品说明书第 7 节均明示。
```

### 核心输出：从"建议"到"统计预演"

不是说"你应该投这几家"，是说：

> **"在这数十次平行宇宙里，你拿到 offer 的概率 98%，最可能的去向分散在声剧-γ、鲸火-1、云枢等多家，中位薪资 53 万，平均握 5 个 offer。**
> **如果你把项目含金量再提升 15 分（相当于多做一个有深度的项目），**
> **平均 offer 数从 5.3 涨到 9.4，中位薪资从 53 万涨到 62 万。"**

这是**统计意义上的"后悔药"**——不是教你怎么做，是让你**看到不做的代价**。

### 同一个引擎，三层可实验视角（已实现 → 可扩展）

我们不是只做"个人求职"，而是一个可实验的人才市场，天然支持三层实验。**这三层共用同一套模拟引擎，第一层已落地、演示视频中完整呈现，后两层是同引擎的自然扩展**：

| 层 | 实验问题 | 现状 |
|---|---|---|
| **个人路径**（已实现） | "我改简历 / 换路线，结局怎么变？" | 反事实滑块 + 数十次真实 sim，**视频中可见** |
| **企业策略**（已有底座） | "只招高学历 vs 招潜力培养，几年后谁更优？" | 已有约 300 家企业分身（门槛/文化/招聘偏好），扩展即可模拟策略对比 |
| **市场趋势**（已有底座） | "某技能贬值 / 新岗位出现后，人才如何重新流动？" | 已有市场看板（约 300 公司 / 约 2000 求职者 / 门槛分布 / 去向流动），扩展即可注入趋势变量 |

> 关键：后两层不是 PPT 空想——**企业分身和市场看板都已真实实现**（演示视频中可见），只是当前默认展示「个人视角」。这让「可实验的人才市场」从愿景变成有据可演的路线图。

## P6 · 核心创新 2 + 3：载体反转 + 建议→预演

### 载体反转：从 PDF 到可对话化身

| 传统 | 我们 |
|---|---|
| 简历 = 静态 PDF，发出去就石沉大海 | 数字分身 = 可被采访的 AI 化身 |
| HR 看 1 分钟，看不深 | HR Agent 可与化身"展开 N 轮持久对话"（chat history 注入，最近 20 轮记忆，能回顾、能总结，不是一问一答的单次 prompt） |
| 候选人对 HR 不透明 | 化身可"现场被采访"，评委/用户问任意问题 |
| 评分纯看简历 5 维 | **学校档（清北/C9/985/211/双非/海外/专升本/专科）+ 学历档（博士/硕士/本科/专升本/专科/高中）双维度加成接入综合分**——专科/专升本下沉群体也能得到精细化建议 |

### 建议→预演

| 传统 AI 求职 | 我们 |
|---|---|
| "你简历项目部分写得不够清晰，建议加 STAR 法则" | **"如果你把项目含金量提升 15 分，多次 sim 里平均 offer 数从 5.3 → 9.4，中位薪资从 53 万 → 62 万。反之简历质量降 20 分，offer 数掉到不足 3 个、薪资回落到 48 万。"** |
| 建议是模糊的"你应该做 X" | 预演是"做 X 后 N 次模拟的统计结果" |
| 没有可量化的反馈 | 滑动条调三个维度，实时看影响 |

这是从"知识传授"到"决策科学化"的跃迁。

### 关键技术点

| 模块 | 技术选型 | 设计取舍 |
|---|---|---|
| **LLM 抽象层** | tier 路由（PRIMARY/SECONDARY/BACKGROUND）+ OpenAI 兼容协议 | 切换模型 = 改 .env 一行，业务代码 0 行变动。覆盖大陆主流模型 |
| **Multi-Agent** | 3 类 LLM Agent + 1 规则市场模拟器：CandidateAgent（求职者分身）/ CompanyHRAgent（公司 HR，注入 hidden_signals）/ InterviewerAgent（面试官，分轮考评）/ CompetitorSimulator（约 2000 人竞争者池，纯规则不调 LLM） | 自研轻量 Multi-Agent 编排（asyncio + 13 周固定 tick 循环）。不用 LangGraph 是因为我们 pipeline 是固定的，多一层抽象反而 debug 难 |
| **反事实分析** | baseline + mutation 变体真实重跑 + 统计聚合 | 每个 mutation 都真跑数十次代价高，我们对关键反事实（项目 +15 / 简历 -20）做真实重跑校准，其余用统计模型插值 |
| **持久化** | JSON 文件 + threading.Lock + 原子写（tempfile → rename） | 约 300 公司 + 约 2000 persona 规模 JSON 全文仍在 MB 级，重读可接受，不上 SQLite 减少复杂度 |
| **动态市场** | admin CRUD endpoints + 数据热更新 | 公司/求职者随时加入退出，sim 启动 snapshot 最新数据 |
| **合规** | sanitizer 双层（prompt 约束 + 落盘前扫描真名）+ 全 synthetic 数据 | 零 PII，全代号公司 |

### 我们刻意**不**做的事

| 不做 | 为什么 |
|---|---|
| LoRA 微调 | 动态市场场景下 LoRA 是反模式——训完就过期，无法跟随公司/求职者池变化 |
| 真实公司数据 | 合规 0 风险优先。BP 此处反复强调 |
| LangGraph 等通用编排框架 | 13 周固定 tick 循环不需要"路径不确定的协作"，自研 asyncio 编排已经够用，多一层抽象反而 debug 难 |
| 声音克隆 / 区块链 / 元宇宙 buzzword | 与产品价值无关，评委一看就识破 |

## P8 · 商业价值：一个市场，多条变现可能

> "可实验的人才市场"更像一层基础设施——现阶段我们**不急于把盈利模式钉死**。这里讲清价值在哪、有哪几条可能的变现路，以及我们此刻（而非最终）的方向判断。早期把账算得太具体，反而是不真诚。

### 这个市场为什么有价值

人才市场的决策——个人怎么投、高校怎么做就业指导、企业怎么定招聘策略——长期只能凭经验。一旦有了可预演、可反事实的沙盘，三方都有动机为"少犯错、看得清"买单。价值不在某个单点工具，而在**让人才市场的决策第一次变得"可实验"**这件事本身。

### 变现的几条可能路径（不预设唯一答案）

| 主体 | 可能的价值 | 现阶段定位 |
|---|---|---|
| 个人 C | 求职决策预演 | 更可能是获客入口 + 数据源，而非主要收入 |
| 高校 G | 群体人才洞察（竞争力分布 / 技能缺口 / 对接雇主） | 有就业率考核与预算，**我们判断可能最先跑通** |
| 企业 B | 反向品牌、招聘策略预演 | 第二曲线 |
| 平台 | 人才洞察数据层、集成 / 分润 | 想象空间 + 被整合的退出可能 |

### 我们此刻的判断（是假设，不是结论）

若要选一个**最先验证**的方向，我们倾向高校——付费动机（就业率 KPI）、预算、决策链都相对清晰。但这是待市场验证的假设，不是已经算清的账。真正可持续的东西是**数据飞轮**：用的人越多，市场模拟越准，洞察越值钱。

### 价值放大：数据飞轮

```
用的人越多 → 市场供需数据越厚 → 模拟/洞察越准 → 对各方越有价值 → 更多人用
```

### 最自然的落地：我们的引擎 × 平台的数据

现在的 demo 是约 300 家虚构公司的沙盘——它的作用是**证明引擎跑得通**。要看清:我们的核心资产不是数据量，是**让人才市场"可实验"的引擎与方法论**。

一旦接入平台级的全量真实招聘数据与算力（如智联这样的招聘平台），这套引擎就能从 demo 沙盘泛化成**覆盖真实市场的人才市场数字孪生**——规模和真实性都会质变。

这也正是**最自然的商业合作形态**：我们提供方法与引擎，平台提供数据、算力与场景。**"缺数据"对我们不是短板，而是与平台合作的接口。** 技术上全量规模用分层架构（全量粗筛 + top 候选精模拟）承接，不是无脑全量 LLM，成本可控。

> 一句话:**我们的框架 × 平台的数据 = 一个能落地的全量人才市场。这就是我们想找的商业合作。**

## P9 · 数据合规（评委关键关注点）

### 我们所有数据都是合成的

| 数据类型 | 来源 | 合规处理 |
|---|---|---|
| 约 300 家虚构公司画像 | LLM 生成 + 模板填充 | 代号化（A 厂 / AI-α / 金-2 等），不挂任何真公司名 |
| 约 2000 个求职者 persona | LLM 生成 | 化名（"张同学"等），零真实信息 |
| HR Agent 行为 | 公开 JD 风格 + 公开面试题库泛化 | 不引用任何真实 HR 行为案例 |
| 用户上传材料 | 用户自己提供 | 落盘在 VM 本地（`backend/data/users/<user_id>/`），**不上云、不训练任何模型**，demo 后可即时删除 |

### sanitizer 双层兜底

```
Layer 1 - prompt 约束：
  "不准出现任何真实公司名（包括 inspired_by_hint）。
   用'某头部短视频平台量级'这种泛指。"

Layer 2 - 落盘前扫描：
  字典匹配 47 条真公司名（字节/腾讯/阿里/华为/...）+ 顶尖院校名
  发现立即替换为代号
```

实际效果：少数公司在 LLM 生成时漏写了真名，**全部被 sanitizer 在落盘前替换**。

### 数据合规声明（提交材料独立 1 页）

我们承诺：

1. 用户上传的简历落盘在 VM 本地，不上云、不训练任何模型，demo 后可即时删除
2. 不收集 PII（个人身份信息）
3. 不向第三方共享任何数据
4. 沙盘里所有约 300 公司 + 约 2000 求职者全部为合成数据，零真人参与

## P10 · 市场策略与增长路径

**为什么是现在（Timing）**：1158 万应届生（教育部 2024）涌入市场、就业率/就业质量考核趋严、高校就业服务数字化采购上升——to G 的求职决策与人才洞察工具正当其时。

### Phase 1（0-6 个月）：标杆校跑通 + C 端获客

- C 端：免费 Web + 小程序，校内口碑 + 求职博主传播，积累使用数据
- G 端：拿下 3-5 所强就业导向标杆校（就业中心试点），打磨群体洞察报告
- **目标**：5 所付费标杆校 + 5 万学生使用

### Phase 2（6-12 个月）：就业口渠道复制 + 企业第二曲线

- 用标杆校案例 + 教育口渠道（就业指导协会、校招联盟）规模化获客
- 启动 B 端企业反向品牌报告（年校招 500+ 企业）
- **目标**：50-100 所付费校，ARR 1000-1500 万

### Phase 3（12-24 个月）：平台集成 + 资本化

- 与招聘平台（含智联）谈集成/分润，作为人才洞察数据层
- 退出路径：被招聘平台或教育就业服务集团收购
- **目标**：300+ 校覆盖，进入 A 轮 / 战略并购

### 北极星指标

- 主战场：付费校数 + 续费率（嵌入就业流程后的粘性）
- 获客层：学生使用渗透率（单校内用过的应届生占比）
- 数据飞轮：模拟准确度随数据量提升的曲线

## P11 · 怎么验证这些假设（而非纸面算账）

> 上一页的商业方向是假设。早期项目真正该做的，是找证据验证它，而不是算一笔看起来精确、其实全靠拍脑袋的账。

### 验证路径

- **先找 3-5 所强就业导向院校做试点**：看真实的付费意愿、愿意为哪部分洞察付费、续不续费——用真实反馈校准方向，而不是预设单价和成本。
- **C 端免费开放**：观察校内自然渗透率与数据积累速度，验证数据飞轮转不转得起来。
- **被验证后再谈规模化**：某个方向跑通了，定价、成本、复制路径那时才有讨论的意义。

### 先验证谁（目标画像）

- **高校（先）**：就业率考核压力大、生源去向分散的强就业导向院校（理工 / 财经 / 综合类一本、双一流）。
- **企业（后）**：年校招 500+、需要"年轻人怎么看我"洞察的中大型科技 / 快消 / 金融 HR。

### 可证伪的预测效度验证设计（下一步计划，非已完成）

> 上面验证的是"商业假设"（有没有人愿意付费）。但更根本的问题是：**模拟引擎本身预测得准不准？** 一个从没被验证过预测力的模拟器，本质上是占卜。我们承认现在**没有**这个验证，但设计了一条可证伪、可执行的验证路径：

- **做法**：找一批**已经毕业、去向已知**的往届生（有明确 offer 记录/最终去向），把他们的简历/背景脱敏后输入引擎，让引擎在**不知道真实结果**的情况下盲跑春招模拟，产出"预测去向分布 / 预测 offer 率"。
- **比对指标**（哪怕小样本也先定义清楚）：
  - **offer 率吻合度**：引擎预测的"拿到 offer 概率"与该往届生真实是否拿到 offer 的二分类命中率（哪怕先做 30-50 人的小样本，也能算出一个 accuracy / AUC）。
  - **去向层级吻合度**：预测的公司梯队（如"头部大厂 / 腰部 / 中小厂"分层)与真实签约公司梯队是否一致，用分层准确率衡量，而不是要求精确命中某一家虚构公司。
  - **薪资区间吻合度**：预测薪资分布的中位数/区间与真实签约薪资的偏差幅度。
- **诚实预期**：约 300 家虚构公司与真实市场不是一一映射，第一轮回测大概率暴露偏差——这恰恰是有价值的信号，用于反向校准 hidden_signals 的分布参数和敏感度系数，而不是用来证明"我们已经很准"。
- **现状**：这是**下一步验证计划**，尚未执行，本次提交没有回测数据可展示。写在这里是为了把"凭什么信模拟能预测真实市场"从一个无法回答的问题，变成一条有明确指标、可执行、可能被推翻的验证路径。

### 一句话

现阶段我们要的，是"把假设放进真实市场验证"的机会——不是一份精确到小数点、却经不起追问的财务预测。

## P13 · 风险与缓解

| 风险 | 缓解策略 |
|---|---|
| **数据合规**（评委 #1 关注） | synthetic 数据 + sanitizer 双层 + 用户材料落盘 VM 本地不上云（已实现） |
| **LLM 成本爆炸** | tier 路由（PRIMARY Claude/DeepSeek，群演 Qwen-Turbo），单次 sim < 0.1 元 |
| **sim 真实性质疑** | 复赛/决赛阶段接入公开真实公司数据建模 + 第三方业内人士背书 |
| **用户上传简历隐私** | 落盘 VM 本地，不上云、不训练任何模型，demo 后可即时删除 |
| **同类竞品快速抄袭** | 工程深度（Multi-Agent + 反事实 + 动态市场）+ 持续迭代 + 早期抢占用户心智 |
| **招聘平台抵触合作** | 我们的价值是"导流 + 提质"，不抢平台核心业务（投递撮合），合作而非竞争 |
| **应届生付费意愿低** | 春招/秋招前后 4 周是付费高峰，其余时间 freemium 留存。决策咨询是高客单补充 |
