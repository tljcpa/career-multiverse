## 一、产品定位（一句话）

**可实验的人才市场沙盘**：把人才市场从"只能凭经验决策"，变成"可以先实验再决策"——让个人、高校、企业在进真实市场前，先看不同选择会演化出什么未来。

第一个落地场景是应届生春招：让 AI 替你跑完 1000 次平行宇宙的春招，先看完结论再决定怎么投递，而不是投出 60 份简历 70% 没回音。

## 二、核心痛点 + 三个反转

### 痛点：人才市场至今只能凭经验决策

天气能预报、城市有数字孪生，唯独人才市场缺一个"先实验、再决策"的地方：
- **个人**凭感觉投递——每年 **1158 万**应届生（教育部 2024），人均投 **60+ 份**（牛客网调研），**70% 没回音**；HR 评价、隐性门槛、三年后是否后悔，全是黑盒猜。
- **高校**凭经验做就业指导，看不清本校学生在市场上的真实竞争力分布与技能缺口。
- **企业**凭直觉定招聘策略，不知道不同策略几年后的人才质量与成本差异。

个人是入口，高校和企业是真正的受益方。

### 我们做的三个反转

**1. 结构反转**：现有 AI 求职工具都帮你做"某个环节"（改简历 / 模拟面试），我们让 AI 替你跑**完整春招**，1000 次——并把它做成一个可重复实验的市场，而不是一次性匹配。

**2. 评估反转**：求职者从一份静态 PDF 简历，变成**LLM 五维评估 + 学校档判定 + 学历加成**的可推理画像；企业同样有"数字分身"（门槛 / 文化 / 招聘偏好）。

**3. 决策反转**：不再是"AI 给你建议"，而是给你 **1000 个平行宇宙的统计结论 + 反事实分析**——从"预测未来"变成"实验未来"：改变某个条件，看结局怎么演化。

### 同一引擎，三层可实验视角（个人已落地，企业/市场已有底座）

| 层 | 实验问题 | 现状 |
|---|---|---|
| 个人路径 | 改简历 / 换路线，结局怎么变 | **已实现，视频中演示** |
| 企业策略 | 招高学历 vs 招潜力，几年后谁更优 | 已有 49 家企业分身，扩展即可 |
| 市场趋势 | 某技能贬值 / 新岗位出现后，人才如何流动 | 已有市场看板，扩展即可 |

后两层不是空想——企业分身和市场看板都已真实实现（视频中可见），只是默认展示个人视角。

## 三、用户旅程（4 步，5 分钟跑完）

```
[1] 上传简历 + GitHub + Blog（或点"使用 Demo 数据"评委体验）
    ↓
[2] LLM 一次性多任务评估
    - 抽取基本字段（姓名、学校、专业、目标岗位、学历）
    - 评估 5 维内部画像（项目 / 实习 / 成就 / 沟通 / GPA 分位，0-100 分）
    - 每维生成评分理由（LLM 引用简历具体证据）
    - 学校档判定（清北 / C9 / 985 / 211 / 双非 / 海外 / 专升本 / 专科）
    - 综合分计算 = (项目+实习+成就)/3 + 学校档加成 + 学历加成 + 沟通修正
    ↓
[3] 数字分身进入 3D 沙盘宇宙（/sandbox 页）
    - 49 家虚构公司按行业聚成"星团"
    - 每家公司有独立 HR Agent（注入 hidden_signals + LLM 真对话）
    - 可采访任意一家 HR：质问加班 / 35+ 占比 / 学历门槛（视频中演示）
    - 点击公司节点触发投递粒子流 + HR 弹层
    - 时间轴 13 周自动推进，沙盘上实时显示投递事件
    ↓
[4] 1000 次平行春招报告（/report）
    - 4 KPI 卡：offer 率 / 签约率 / 平均 offer 数 / 中位数年薪
    - 4 数据图表：Offer 分布 / 接 offer 周次 / 公司 offer Top 15 / 最终去向饼图
    - 决策树：本次 sim 真实 journey 可视化
    - 反事实滑动条：拖 3 个维度看数字真实变化
    - AI 教练给的关键结论：LLM 根据 5 维分实时生成的可执行建议
```

附加页面：
- `/admin` 市场治理：可现场加一家公司或求职者，沙盘立刻更新节点（视频中演示）
- `/dashboard` 市场看板：全市场 KPI + 行业分布 + 招聘门槛分布 + 学校 tier 分布

## 四、技术架构

### 后端
```
FastAPI 0.115 + Uvicorn (Python 3.10)
├── LLMRouter（tier 路由：PRIMARY / SECONDARY / BACKGROUND）
│   └── OpenAI 兼容协议，适配 DeepSeek / Qwen / Kimi / GLM 等
├── Multi-Agent 沙盘引擎（自研轻量编排，不用 LangGraph）
│   ├── CandidateAgent（求职者分身，LLM 决策）
│   ├── CompanyHRAgent（注入 hidden_signals，含 5 个文化标签 + 隐性门槛）
│   ├── InterviewerAgent（分轮考评，LLM 评分）
│   └── CompetitorSimulator（200 人竞争者池，纯规则模拟避免成本爆炸）
├── 反事实分析（aggregator + monotonic_step 单调插值）
├── pypdf 简历真解析
├── SessionStore（in-memory + threading.Lock）
└── HR 多轮对话历史（per (user_id, company_code) keyed，20 轮 trim）
```

## 五、关键技术亮点

### 1. LLM 评估透明化（反"黑盒"设计）
- 每个分数都有 LLM reasoning（引用简历具体证据，如"GitHub 2.5k star + Flink+Redis 千万级 QPS"）
- Profile 页 "▾ AI 评分依据" 折叠展开看判定
- "▾ 查看评分构成" 折叠展开看综合分 88 = 66 + 15（学校档）+ 5（学历）+ 2（沟通）
- "ℹ 评分参考标准" 弹层看完整 rubric（70+/40-65/<30 区间）

### 2. 故事-实现一致性
- 不吹 LoRA：动态市场场景下 LoRA 是反模式（注释明确），改用 LLM 一次性多任务评估
- 不吹 LangGraph：固定 pipeline 不需要"路径不确定的协作"
- 透明披露："本次基于 3 次真实 LLM-Multi-Agent sim + bootstrapping 扩展到 1000 个"
- 不撒谎"1000 次蒙特卡洛"——前端显著位置说明真实跑了 3 次 + 统计扩展

### 3. 反 prompt injection + 数据安全
- 简历内容用 `<<<RESUME_START>>>` 分隔符包裹喂 LLM
- 输入端过滤防"关闭分隔符"攻击
- HR system prompt 加铁律防 emoji 包装泄露具体百分比
- path traversal 防御：上传文件名 sanitize（防 `../../../etc/cron.d/x`）
- API rate-limit 由 Pydantic 边界约束兜底（n_runs ≤ 10000 / question ≤ 500 字符）

### 4. 动态市场（视频中现场演示）
- /admin 加公司 / 改公司 / 删公司 → 下次 sim 立刻生效
- 同一 user 多次 sim 互不干扰
- nginx 反代 + atomic write（tempfile + rename）保证文件并发安全

## 六、合规说明

- 全部 49 家公司均为**虚构代号**（鲜锋 / 焰火 / 量-π / 圈层-α 等），不影射真实公司
- 200 名虚构竞争者持有合成 persona，零 PII
- sanitizer 字典内置 47 条真公司名（字节 / 腾讯 / 阿里 / 华为等）+ 顶尖院校名，落盘前扫描替换
- 评委上传的真简历仅落盘到 `backend/data/users/<user_id>/`（VM 本地），不上云、不训练任何模型、demo 后可手动清理
- 数据来源全部公开 / 合成，零合规风险（详见数据合规声明）

## 七、产品差异化 vs 现有 AI 求职工具

| 维度 | 现有工具（简历优化器 / 面试模拟器） | 春招平行宇宙 |
|---|---|---|
| 解决环节 | 单点（投递前 / 面试中） | 完整春招全流程 1000 次 |
| 评估依据 | 简历文字关键词 | LLM 五维评分 + 评分理由 + 学校档 + 学历加成 |
| 输出 | 建议 / 分数 | 统计结论 + 反事实预演 + 个性化教练建议 |
| 现场可操作 | 测自己简历 | 改公司 / 改求职者 / 和虚拟 HR 对话（视频中演示） |
| 评分透明度 | 黑盒分数 | 3 层展开（评分依据 / 综合分构成 / 评分标准 rubric） |
