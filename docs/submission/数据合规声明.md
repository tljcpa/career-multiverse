## 一、数据全链路

### 1. 公司数据（约 300 家虚构公司）
- **全部为虚构代号**（声剧-γ / 鲸火-1 / 云枢 / 量-π / 圈层-α / 视界-Ω 等），无任何真实公司名称
- 来源：`scripts/collect_companies.py` 用 LLM 生成 + sanitizer 双层过滤
- **sanitizer 内置 47 条真公司名字典**（字节 / 腾讯 / 阿里 / 华为 / 美团 / 拼多多 / 京东 / 蔚来 / 小鹏 / 理想 / OPPO / vivo / 小米 / 大疆 / 商汤 / 旷视 / Google / Microsoft / Meta / Amazon 等），LLM 输出后落盘前**强制扫描替换**为代号
- 存储位置：`backend/data/companies/companies_v1.json`（仓库内随代码公开）
- 字段：code_name / industry / size_label / 9 个 OfficialJD（对齐答疑文档 Q2 推荐格式）/ hidden_signals（hiring_bar / culture_tags / hidden_filters / pct_over_35 / business_growth）

### 2. 求职者数据（约 2000 名虚构竞争者）
- **全部为合成 persona**，无任何 PII（真实姓名 / 身份证 / 手机 / 邮箱）
- 来源：`scripts/collect_personas.py` 用 LLM 生成 + 学校档 / 经历各档分布约束
- 存储位置：`backend/data/personas/competitors_v1.json`
- 字段：candidate_id / official_cv（11 字段对齐答疑文档）/ hidden_signals（5 维评分）

### 3. 评委上传的真简历（demo 期）
- 上传文件落盘到 `backend/data/users/<user_id>/`（VM 本地，**不上云**）
- 不参与任何模型训练，不发送给除 LLM provider 之外的第三方
- 文件名 sanitize（`Path(filename).name`）+ 路径校验防 traversal 攻击
- demo 结束后**可手动清理**（赛事委托方 / 评委可申请删除）
- 简历文本用于本次 LLM 评估调用 + 反事实分析，处理过程中同时存在于内存 SessionStore（供当前会话复用）与落盘文件（`backend/data/users/<user_id>/`，供 demo 后审计/清理）
- 评委如果点 "使用 Demo 数据" 按钮，**零上传**，使用内置王明 persona

### 4. LLM 调用数据流
- 唯一上游：DeepSeek API（`https://api.deepseek.com/v1/chat/completions`）
- 简历内容 → 用 `<<<RESUME_START>>>` 分隔符包裹（防 prompt injection）→ LLM 评估返回 JSON
- LLM 调用日志只记录 token 数 + 耗时（structured logging），**不记录简历内容**
- DeepSeek 隐私政策：https://platform.deepseek.com/privacy（用户数据不用于训练，30 天清除）

## 二、合规要点

### ✅ 满足
- **零真实公司名**：sanitizer 双层 + 全代号
- **零 PII**：约 2000 个虚构 persona + 评委上传简历不上云
- **零未授权数据**：所有公开数据来自答疑文档允许范围
- **可追溯**：数据来源 / 处理流程全部开源在 GitHub
- **可删除**：评委如要求清理上传的简历，10 秒删除（`rm -rf backend/data/users/<uid>/`）
- **可审计**：sanitizer 字典 / LLM prompt / 评分公式全部在仓库内透明，无黑盒
- **Cloudflare Flexible SSL 已启用**，HTTPS 可正常访问。

### ⚠️ 已知限制
- **admin token 默认值 `multiverse-demo-2026` 写在仓库**（仅用于 demo，生产部署需走 `ADMIN_TOKEN` env 变量）
- **LLM tier routing 单 provider（DeepSeek）**，没有自动 fallback（评委如要求换 Qwen，改 1 行 .env 即可）
- **SessionStore in-memory**（无 Redis），VM 重启用户身份丢失，需重新上传简历（不影响数据安全）
