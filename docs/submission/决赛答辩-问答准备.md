# 决赛答辩 · 问答准备(评委问答 6 分钟)

> 7/9 线下答辩,评委问答 6 分钟。下面是高频问题 + 备好的答法。
> 原则:诚实(个人端真、企业学校端标演示)、有据(竞品调研支撑)、不吹逐行原创。

## Q1. 你和现有 AI 求职工具(简历优化 / 面试模拟)有什么区别?(最可能问)

现有产品都是**单边、静态**的:简历优化(Rezi/超级简历)绕 ATS 改词、面试外挂(Final Round AI)实时喂话术、平台匹配(猎聘 AI)是词袋对词袋。它们给你"武器",但回答不了"我该不该投、投了会怎样"。

我们做的是**可实验的沙盘**,三点是别人没做的:
1. **引入竞争者分身**——人人都用 AI 改简历后,市场"简历通胀",我们能算出"别人也优化后,你的相对竞争力"。
2. **多主体博弈 vs 单边匹配**——能模拟面试官偏好、企业隐性门槛这些"招人玄学"。
3. **量化反事实**——不止给一个"简历 80 分",更能回答"补一段大厂实习,offer 率提升多少",把职业规划从凭经验变成基于概率的决策(因果推断)。这一层目前市面上少见。
> 佐证:这类"多主体人才市场模拟 + 反事实"在学术界有探索(WorkSim/Simploy/Life2Vec),但**商业产品几乎空白**。

一句话概括身位差:**当大家还在打磨一把更快的"镰刀"时,我们在造一个"气候模拟器"**——其他产品试图提高单次匹配的成功率,我们在计算整个职业生涯的胜率。工具只能改变效率,只有沙盘,能改变战略。

## Q2. 数据从哪来?是真的吗?(诚实口径,别被戳穿)

分层如实答:
- **个人端是真的**:真实 LLM 五维评估 + 真实 Multi-Agent 模拟,现场可跑给你看。
- **模拟次数**:真实跑数十次(本轮基线 30 次)完整 Multi-Agent 对局、每次跑满 13 周,再用 bootstrap 重采样统计推断出分布——前端进度条明示,**不谎称"真跑 1000 次"**。"平行宇宙"是概念表述。
- **企业端 / 学校端**:是基于同一引擎 + 已有市场数据(约 300 家企业分身 / 约 2000 竞争者)的**演示视图**;学校端数据是"C 端学生个人 sim 结果按学校群体聚合脱敏",不展示单个学生可识别信息(见数据合规声明)。
- 全部公司为**虚构代号**,零真实企业、零 PII。

## Q2b. 我扫码试了,数字和 PPT 里不一样 / 每次跑结果都不同?(把浮动讲成卖点)

这正是产品的核心特性,不是 bug——我们做的是**实时动态的人才市场**,不是查一张静态报表。
- 公司和竞争者在市场里动态进出,每次模拟都是**当下市场的真实演化**,结果自然浮动。
- PPT 里的 offer 率 98.4% 是**某一轮 30 次模拟的统计结论**;你扫码看到的是**当下实时跑的这一次**。就像天气预报每次刷新都在变——因为市场是活的、在动态演化。
- 这恰恰证明它不是写死的 demo,而是一个**真在运转、可实验的动态市场**——你甚至可以现场在治理页加一家公司、加一批竞争者,报告立刻跟着变。

> **节奏提醒(避免现场翻车)**:口播讲结论数字时统一说"约 98%""约 53 万"而不是死抠小数点,现场 demo 的实时数字和 PPT 的历史统计数字本来就不该完全一致——提前用"约"字弱化精确值预期,评委追问时按上面口径接,不要临场去凑数字对齐。

## Q2c. 现场跑的到底是真实在线推理,还是预先录好的结果?

诚实分两种情况,别混着答:
- **现场演示走的是真实在线跑**:上传简历→五维评估→进沙盘→报告,这条链路是真实调用 LLM API 的完整流程,不是预置动画,评委可以现场传自己的简历验证。
- **PPT 里引用的具体数字(98.4% offer 率 / 中位 53 万)来自我们提前跑好、已经过 bootstrap 统计聚合的一轮完整结果**,不是编的,但也不是"现场这一次"的结果——这是两回事:一个是"当下这次真实推理"(现场会浮动),一个是"历史某轮真实推理的统计结论"(PPT 固定引用)。把这两者的关系讲清楚,评委追问"数字对不上"时直接引用 Q2b 的答法。

## Q2d. B 端为什么非要动态多智能体 sim,一份静态灵敏度报告(Excel 拉一下敏感系数)不是更便宜吗?

这题要正面接住,不能只讲"我们技术更炫":
- **静态报告算的是"你动、别人不动"**——比如"起薪降 2000,流失率 +X%",这是**局部偏导**,隐含假设竞争对手按兵不动。
- **真实招聘是动态博弈**:你降薪的同时,同赛道其他公司也在根据市场反应调整 JD、薪资、门槛;供需和政策也在变。你要的不是"我单独变一个变量会怎样",而是**"大家都在变的情况下,我这步棋最后落在哪"**——这是博弈论意义上的均衡,不是敏感度系数能算出来的。
- **举例**:静态表格答不出"如果我降薪 2000,竞对 A 会不会趁机提薪抢人、竞对 B 会不会跟我一起降形成合谋式踩踏",但多智能体 sim 里每个 HR Agent 都有自己的策略和反应,能把这种连锁反应跑出来。
- 一句话:**静态报告给的是"这一脚踩下去多重",动态 sim 给的是"这一脚踩下去,水会怎么晃、别人会不会也跟着踩,最后水面停在哪"。** 这正是这个引擎不可替代、不能被一张 Excel 替代的地方。

## Q3. 企业端 / 学校端能真跑吗,还是 PPT?

诚实:**个人端已完整实现、线上可跑**(现场演示)。企业端和学校端当前是**演示视图**,但**底座已经在线**——约 300 家企业分身(含招聘门槛/文化/筛选策略)、约 2000 竞争者池、市场看板都是真实实现的,只是默认展示个人视角。企业策略实验、学校群体洞察是**同一个引擎的自然扩展**,不是空想。这是"三层可实验市场"的路线图:个人已落地,企业/学校是下一步。

## Q4. 商业模式?怎么赚钱?

先定位:金融投资哪怕只投 10 万,也会先在历史数据里跑一万次回测;但人生最大的一次投资——择业,企业几百万量级的招聘投入,至今还是靠肉身试错。《春招平行宇宙》想成为**人力资本市场的量化回测沙盘**——接入真实数据后,它可以是**招聘界的"彭博终端"**。

诚实答:**现阶段我们不急于把盈利模式钉死**——它是一层基础设施,变现路径是开放的。
- 价值在于:让人才市场的决策第一次变得"可实验",三方(个人/高校/企业)都有动机为"少犯错、看得清"买单。
- 变现有多条可能且已经能讲具体:
  - **C 端**:职业生涯 GPS,卖的是"确定性",更像获客入口 + 数据源;
  - **B 端(想象空间最大的一条)**:企业版是"雇主品牌与薪酬动态定价沙盘"——企业若想把某岗位起薪下调 2000 元,不用真发 JD 被市场毒打,先在沙盘里跑一万次,立刻看到高潜人才流失率会升高几个百分点,这是帮企业省下数百万真实试错成本;
  - **G 端**:高校群体人才洞察,校长拉动反事实滑块看"换一门课,毕业生薪资分布怎么变"。
- 若要选一个**最先验证**的方向,我们判断是高校——有就业率考核、有预算、决策链短。但这是**待市场验证的假设,不是已经算清的账**。
- 早期该做的是找几所标杆校验证真实付费意愿、把数据飞轮转起来,而不是纸面上给一个精确却经不起追问的财务模型。

（若评委追问具体单价/成本:坦诚说"现在给不出可信的具体数字,那需要真实试点验证后才有意义;硬编一个反而不诚实"——这个回答本身就是加分。）

## Q4b. 只有约 300 家公司、跑几十次,会不会太小、太局限?怎么规模化到真实市场?(这题是我们最想讲的)

诚实:约 300 家、跑数十次是 **demo 规模**,作用是证明引擎跑得通。我们真正的资产不是数据量,是**让人才市场"可实验"的引擎与方法论**。
- **规模化路径**:接入真实全量招聘数据 + 算力,这套小沙盘就变成覆盖真实市场的**人才市场数字孪生**。
- **这恰恰是与智联最自然的商业合作形态**:我们出引擎和方法,平台出数据、算力、场景。**缺数据对我们不是短板,是合作的接口。**
- 技术上,全量规模下用**分层架构**(全量粗筛 + top 候选精模拟),不是无脑全量 LLM——规模和真实性会质变,成本可控。
- 一句话对着评委(尤其智联方)讲:**我们的框架 × 你们的数据 = 一个能落地的全量人才市场。这就是我们想找的商业合作。**

（若评委追问"你凭什么信模拟能预测真实市场,现在有没有回测过":诚实答**现在没有**,但有明确的下一步验证路径——找一批**已知真实去向/offer 记录**的往届生,脱敏后让引擎盲跑,比对"预测 offer 率 / 预测公司梯队 / 预测薪资区间"与真实结果的吻合度,哪怕 30-50 人的小样本也能算出可信度指标(如二分类命中率)。这不是已经做完的事,是我们认为在拿到真实数据后第一件要做的验证,而不是先讲故事再回避验证。）

## Q4c. 你沙盘里的约 300 个 HR Agent 是大模型设定的,凭什么信它们的反应和真实智联平台上的 HR 一样?

诚实分层答,不回避:
- **现状**:目前 HR Agent 的偏好完全由 LLM 的先验知识驱动,这是**冷启动**——它们没有被任何一条真实招聘行为数据校准过,本质是"基于常识的合理近似",不是"已验证的真实还原"。
- **规划(尚未实现,标注清楚是路线图)**:我们把这条路径叫**"数字孪生对齐引擎"(Digital Twin Calibration Loop)**——如果接入智联平台每天真实发生的海量行为数据(已读不回 / 主动发起沟通 / 发放 offer 等),理论上可以用强化学习的方式持续校准沙盘里 HR Agent 的偏好分布,让它们逐步逼近真实市场当下的风向。
- **诚实边界**:这一步我们**没有做**,没有任何实验数据支撑效果,今天讲的是方向而不是成果。一旦真正跑通,沙盘的定位会从"孤立的模拟沙盒"升级为"招聘市场的实时数字孪生";在那之前,请把它理解成一个有明确路径、但尚未验证的愿景。

## Q4d. 学生看完"项目 55 分、建议加强"这个结论,然后呢?就给一句建议就完了?

诚实分两半答,别把"已有"和"规划"混着说:
- **已实现的**:AI 教练页不是只给一个分数,是基于该学生的五维分 + 学校档/学历档,LLM 生成"top 强项 / 最大瓶颈 / 3 条可执行建议"(视频中可见),而且能配合反事实滑块——"如果项目分从 55 提到 70,offer 数和薪资分布怎么变",把"你不够好"变成"你差在哪、补到什么程度值多少"。
- **还没做、如实说是规划的**:再往下一步的"资源对接"(比如关联到具体的开源项目练习入口、内推/实习入口),现在**没有实现**,是路线图,不是已完成的闭环。行动闭环目前止步于"看清差距 + 量化建议",还没有接到"具体去哪里补"这一环。
- 为什么诚实讲更有说服力:一个刚做 22 天的 demo 如果说"我们已经打通实习内推",那才是真正会被现场问穿的破绽;老实说"评估→建议已经做实,资源对接是下一步要接的现实世界接口",反而立得住。

## Q4e. 你这约 300 家公司是不是刻意做得很平均、失真了真实市场的竞争格局?

先说实情,不回避:我们查过 `backend/data/companies/companies_v1.json` 的分布——行业维度大量是 8 家/4 家/2 家/1 家这种**人为设定的均衡采样**(比如互联网细分赛道各配 8 家、更细分的新兴方向各配 2-4 家),规模标签也是 LARGE/MEDIUM/SMALL/STARTUP/MEGA 大致均匀分布,不是真实市场那种"头部几家吃掉大部分招聘量"的长尾结构。
- **为什么这么做(demo 阶段的工程选择,不是想造假)**:22 天单人 demo,目标是**覆盖尽量多的赛道和公司规模**,让评委在互联网/AI/金融/硬件/新能源等不同赛道都能看到有代表性的公司,而不是把有限的 300 个位置全砸在少数头部大厂上导致其他赛道空白。均衡采样是为了"广度可演示",不是为了"贴近真实分布"。
- **诚实承认代价**:这确实意味着当前 sim 里的"竞争烈度""offer 概率""中位薪资"是在一个人为拉平的市场里算出来的,和真实市场(头部集中、长尾企业招聘量小得多)的竞争强度不是一回事。**如果评委追问"中位薪资 53 万是不是偏高"**——诚实答:这是在均衡采样的 demo 市场里、且样本本身是我们生成的中高配置画像下跑出的统计结果,不能直接类比真实全量市场的中位数,真实数字需要接入真实招聘数据后重新跑。
- **规模化时怎么改**:接入真实数据后,行业和规模分布要按**真实市场的长尾分布**重构采样权重(头部大厂招聘量大、公司数少;长尾中小企业公司数多、单家招聘量小),而不是继续均衡采样。这是 P11 提到的"回测校准"要连带解决的问题之一。

## Q4e-2. 那个"宝妈绝境案例"是真人真事吗?24 次全 0 offer 靠谱吗?

两层诚实澄清,评委大概率会追问,提前讲清楚比被问到才补更稳:
- **"宝妈"不是真人**:这是我们构造的虚构 persona(合成的典型画像——28岁/双非本科文科/2年全职带娃空窗/目标项目管理),零 PII,不是某个真实求职者的真实数据。她的作用是给出一个有代表性的困难画像,让沙盘真跑、看真实结果,而不是编一个励志故事。
- **这一轮用的是降级模型,结果要打折扣看**:跑这批 sim 时 DeepSeek 账户余额耗尽(402),临时切到 NVIDIA 免费层的 `meta/llama-3.1-8b-instruct`(比 DeepSeek-Chat 弱很多的模型)完成,并且因为免费层 RPM 很低,脚本里加了较重的限流。基线/纯包装/真补实力三档各真跑 8 次、共 24 次真实 LLM 沙盘,raw 结果确实是 0 面试 0 offer,**这不是编的**,但**24 次全 0 这个结果可能部分归因于弱模型筛选偏严苛**,不能排除换成 DeepSeek-Chat 或更强模型跑同一批 sim,offer 率会有所回升。
- **所以我们怎么讲这个案例**:不说"事实证明双非文科转岗必死",而说"这轮沙盘推演揭示的方向是赛道错配"——弱模型 + 小样本(24 次)的结果只能支撑一个方向性判断,不能当成精确结论。这个诚实限定必须讲,省略掉就是过度宣称。

## Q4f. 同一份简历评分稳不稳定?换一个 LLM 分数会不会差很多?

诚实分"已经处理的坑"和"没做过的验证"两半答:
- **已经踩过、已经修的坑**:五维评分这一步(_extract_resume_summary)最初用的采样温度是 0.3,实测同一份简历重复上传综合分会抖动 ±5 分——这在评委现场复测时是会穿帮的硬伤,所以已经把这一步的 temperature 降到 0.1(代码里有明确注释说明原因:评分/学校档判定是"追求一致性"的任务,不是创意生成)。这不是没做过测试,是真实测出问题后改的。
- **诚实边界,没有回避**:temperature=0.1 不是 0,理论上仍有小概率的分数漂移,而且我们**没有做过系统性的"同一简历重复 N 次测波动方差"的量化实验**,也**没有测过换一个 LLM(比如从 DeepSeek 换成 Qwen)分数一致性如何**——这两项都是待验证,不是已经验证过没问题。
- **具体、可证伪的验证设计(下一步要做的,不是已完成)**:
  - **同模型稳定性**:取 20-30 份代表性简历,每份用同一 LLM 重复评估 10 次,统计综合分的标准差,定一个可接受阈值(比如 ±2 分内算稳定),超过就说明 temperature 或 prompt 还需要收紧。
  - **跨模型一致性**:同一批简历分别用 DeepSeek / Qwen / GLM 跑一遍,比较五维分排序的相关系数(如 Spearman),排序高度一致即使绝对分有偏移也可接受;如果排序都对不上,说明评分逻辑过度依赖某个模型的先验,需要重新设计 rubric prompt。
  - 这两项和 P11 的"预测效度回测"是同一批要做的"上线前必须过的可信度关卡",不是锦上添花。

## Q5. 技术壁垒 / 护城河?

三层:(1) **数据飞轮**——用的学校/学生越多,市场供需数据越厚,模拟越准,洞察越值钱;(2) **多主体 sim 引擎**——自研轻量 Multi-Agent 编排,不是简单调 API;(3) **反事实因果推断**——把"改一个变量看结局分布"做成产品,竞品没有。早期标杆校 + 就业口渠道形成先发壁垒。

如果评委追问"这些壁垒是不是也能被抄":可以用身位差回答——**工具类护城河容易被追平(改简历这把镰刀谁都能磨快),但沙盘类护城河很难被追平,因为它靠的是数据飞轮和模拟准确度的复利,不是单点功能。别人抄得走界面,抄不走跑了多少轮真实数据校准出来的模拟精度。**

## Q6. 团队情况?(诚实,反而是加分)

非计算机科班,单人 22 天,AI(Claude Code)当队友做工程实现——核心创意、产品方向、架构决策由本人主导。这本身就是 AI 应用赛道的范本:一个人 + AI 把高门槛工程做出来。落地证据:真实录屏演示 + 开源代码(可本地部署) + 线上可访问 demo。

## Q7. 为什么用 DeepSeek 不做模型微调 / LoRA?

动态市场场景下 LoRA 是**反模式**:公司和求职者随时进出,微调出来的模型今天训练明天就过期。我们用 LLM 一次性多任务实时评估,永远基于最新的人和最新的市场。这是刻意的架构取舍,不是没能力做。

## Q8. 合规 / 隐私?

全部约 300 家公司虚构代号、约 2000 竞争者合成 persona,零 PII;评委上传的真简历仅落盘本地(不上云、不训练任何模型、demo 后可清理);学校端群体聚合脱敏。详见《数据合规声明》。
