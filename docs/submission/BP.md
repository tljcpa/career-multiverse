## P2 · 应届生求职的真问题

每年 **1158 万**应届生（2024 教育部数据）涌入就业市场。**平均每人投出 60+ 份简历**（牛客网年度调研），**70% 没有回音**。

但真正让应届生焦虑的，不是投递数量——是**三个看不见**：

### 看不见 1：HR 的真实评价
你投出去的简历，HR 用 1 分钟扫过，淘汰理由是什么？是学校？是项目深度？是 keyword 不匹配？没人告诉你。下一次怎么改？只能猜。

### 看不见 2：公司的隐性门槛
JD 写"本科及以上"，实际录取者 92% 是 211 起步。JD 写"不限性别"，实际同岗位男女比是 7:3。JD 写"35 岁以下"是违法，但实际筛简历时年龄是第一个被看的字段。**这些 JD 不写但人人都用的隐性维度**，应届生进不了"局内人视角"。

### 看不见 3：决策的后悔成本
拿到字节 + 美团两个 offer，选哪个？三年后你会怎样？你的同学选了另一个，他三年后怎样？没人能告诉你**反事实**。决策只能赌一次，且永远没法验证。

**现有 AI 求职工具解决的是"投递效率"这种局部问题**——简历优化、面试模拟、JD 匹配。**但应届生真正需要的是上面三个"看不见"的解药**。

### 往上看一层：这不只是个人的问题

天气可以预报、城市可以用数字孪生优化，但**人才市场至今只能靠经验决策**——个人凭感觉投递、高校凭经验做就业指导、企业凭直觉定招聘策略。整个市场缺一个"先实验、再决策"的地方。我们想做的，就是**第一个可实验的人才市场**：个人是入口，高校和企业是真正的受益方。

## P3 · 解决方案：把整个春招"虚拟跑一遍"

### 用户体验流程（4 步）

```
[1] 上传简历 + GitHub + Blog
        ↓
[2] AI 一次性多任务评估你的简历（抽取核心字段 + 5 维内部画像评分 + 每维评分理由 + 学校档判定 + 学历加成）
        ↓
[3] 数字分身进入沙盘宇宙：
    - 49 家虚构公司（按行业聚成"星团"，每家有独立 HR Agent）
    - 200 个竞争者池（你的真实竞争对手画像）
    - 模拟 13 周完整春招
    - 并行跑 1000 次（蒙特卡洛）
        ↓
[4] 输出：
    - 1000 次平行宇宙的 KPI 统计
    - 各公司 offer 概率分布
    - 决策树（每次 sim 的关键岔路）
    - 反事实分析（"如果你改 X，结果如何变化"）
    - "采访 HR"：可提问任意公司的 HR Agent（**支持多轮持续对话，会话级 chat history 持久化，不是一问一答**，演示视频中展示）
    - **AI 教练个性化建议**：基于本人 1000 次 sim 统计 + 5 维评分 + 学校档/学历档，LLM 给出"top 强项"/"最大瓶颈"/"3 条可执行建议"

> **关于"1000 次"的真实采样架构（透明披露）**：每次启动 sim 实际完整跑 **3 次 LLM Multi-Agent 真实对局**（每次约 30 个 LLM 调用，5 类 agent × 多轮交互），然后用 bootstrap 重采样 + 蒙特卡洛扰动扩展到 1000 个统计样本——平衡延迟（用户等 60-150 秒 vs 真跑 1000 次需 1+ 小时）和统计稳定性。前端进度条文案 + 产品说明书第 7 节均明示。
```

### 核心输出：从"建议"到"统计预演"

不是说"你应该投这几家"，是说：

> **"在 1000 次平行宇宙里，你最可能的去向是焰火（40% 概率），中位薪资 49 万。**
> **如果你把项目含金量再提升 15 分（相当于多做一个有深度的项目），**
> **平均 offer 数从 3.7 涨到 4.5，平均薪资 +14 万。"**

这是**统计意义上的"后悔药"**——不是教你怎么做，是让你**看到不做的代价**。

### 同一个引擎，三层可实验视角（已实现 → 可扩展）

我们不是只做"个人求职"，而是一个可实验的人才市场，天然支持三层实验。**这三层共用同一套模拟引擎，第一层已落地、演示视频中完整呈现，后两层是同引擎的自然扩展**：

| 层 | 实验问题 | 现状 |
|---|---|---|
| **个人路径**（已实现） | "我改简历 / 换路线，结局怎么变？" | 反事实滑块 + 1000 次 sim，**视频中可见** |
| **企业策略**（已有底座） | "只招高学历 vs 招潜力培养，几年后谁更优？" | 已有 49 家企业分身（门槛/文化/招聘偏好），扩展即可模拟策略对比 |
| **市场趋势**（已有底座） | "某技能贬值 / 新岗位出现后，人才如何重新流动？" | 已有市场看板（49 公司 / 200 求职者 / 门槛分布 / 去向流动），扩展即可注入趋势变量 |

> 关键：后两层不是 PPT 空想——**企业分身和市场看板都已真实实现**（演示视频中可见），只是当前默认展示「个人视角」。这让「可实验的人才市场」从愿景变成有据可演的路线图。

```

零和、不透明、不可逆。

```

不是优化"投递"这个动作，是**重构"投递→筛选"的市场结构**——把一次性的赌博，变成可重复的统计预演。

## P6 · 核心创新 2 + 3：载体反转 + 建议→预演

### 载体反转：从 PDF 到可对话化身

| 传统 | 我们 |
|---|---|
| 简历 = 静态 PDF，发出去就石沉大海 | 数字分身 = 可被采访的 AI 化身 |
| HR 看 1 分钟，看不深 | HR Agent 可与化身"展开 N 轮持久对话"（chat history 注入，最近 20 轮记忆，能回顾、能总结，不是一问一答的单次 prompt） |
| 候选人对 HR 不透明 | 化身可"现场被采访"，评委/用户问任意问题 |
| 评分纯看简历 5 维 | **学校档（清北/C9/985/211/双非/海外/专升本/专科）+ 学历档（博士/硕士/本科/专升本/专科/高中）双维度加成接入综合分**——专科/专升本下沉群体也能得到精细化建议 |

### 建议→预演

| 传统 AI 求职 | 我们 |
|---|---|
| "你简历项目部分写得不够清晰，建议加 STAR 法则" | **"如果你把项目含金量从 65 提到 80，1000 次 sim 里 offer 数从 3.7 → 5.2，薪资 +14 万。如果继续提到 90 收益递减。"** |
| 建议是模糊的"你应该做 X" | 预演是"做 X 后 N 次模拟的统计结果" |
| 没有可量化的反馈 | 滑动条调三个维度，实时看影响 |

这是从"知识传授"到"决策科学化"的跃迁。

### 关键技术点

| 模块 | 技术选型 | 设计取舍 |
|---|---|---|
| **LLM 抽象层** | tier 路由（PRIMARY/SECONDARY/BACKGROUND）+ OpenAI 兼容协议 | 切换模型 = 改 .env 一行，业务代码 0 行变动。覆盖大陆主流模型 |
| **Multi-Agent** | 3 类 LLM Agent + 1 规则市场模拟器：CandidateAgent（求职者分身）/ CompanyHRAgent（公司 HR，注入 hidden_signals）/ InterviewerAgent（面试官，分轮考评）/ CompetitorSimulator（200 人竞争者池，纯规则不调 LLM） | 自研轻量 Multi-Agent 编排（asyncio + 13 周固定 tick 循环）。不用 LangGraph 是因为我们 pipeline 是固定的，多一层抽象反而 debug 难 |
| **反事实分析** | baseline + N mutation 变体，monotonic 单调插值 | 真跑 1000 次每个 mutation 不现实（成本爆炸）。统计模型 + 真 sim 校准 |
| **持久化** | JSON 文件 + threading.Lock + 原子写（tempfile → rename） | 49 公司 + 200 persona 规模 JSON 全文 < 1MB，每次重读 50ms 可接受，不上 SQLite 减少复杂度 |
| **动态市场** | admin CRUD endpoints + 数据热更新 | 公司/求职者随时加入退出，sim 启动 snapshot 最新数据 |
| **合规** | sanitizer 双层（prompt 约束 + 落盘前扫描真名）+ 全 synthetic 数据 | 零 PII，全代号公司 |

### 我们刻意**不**做的事

| 不做 | 为什么 |
|---|---|
| LoRA 微调 | 动态市场场景下 LoRA 是反模式——训完就过期，无法跟随公司/求职者池变化 |
| 真实公司数据 | 合规 0 风险优先。BP 此处反复强调 |
| LangGraph 等通用编排框架 | 13 周固定 tick 循环不需要"路径不确定的协作"，自研 asyncio 编排已经够用，多一层抽象反而 debug 难 |
| 声音克隆 / 区块链 / 元宇宙 buzzword | 与产品价值无关，评委一看就识破 |

## P8 · 商业模式：主战场是高校，不是三边通吃

> 我们不做"三边都收钱"的空想。一个清晰的主战场 + 两条扩展曲线，比平铺三个买家更经得起追问。
> **主战场 = 高校就业指导中心（to G / 校园 B 端）。C 端是获客与数据源，企业 + 招聘平台是第二曲线与资本故事。**

### 主营收：高校就业指导中心（to G 订阅）

**为什么是高校，不是应届生个人**：

| 维度 | C 端应届生 | 高校就业中心（主战场） |
|---|---|---|
| 付费能力 | 学生穷，付费率低 | 有年度就业服务采购预算 |
| 刚需驱动 | 凭兴趣，非刚需 | 教育部就业率/就业质量考核是硬 KPI |
| 留存 | 春招 2-4 月用完即走 | 嵌入就业指导流程，长期续费 |
| 获客 | 千万个体，单客获客贵 | 找到就业中心主任即可成单 |

**学校买的不是"帮单个学生预测 offer"，而是群体级人才洞察**——这正是"可实验的人才市场"对学校的价值：
- 本校学生在市场上的竞争力分布、与哪些雇主匹配
- 专业级技能缺口（学生群体短板维度）、应重点对接的雇主
- 把就业指导从"老师凭经验谈话"升级为"数据驱动的群体决策"

**收费方式**：按校年度订阅（SaaS），客单价随在校生规模与功能档分级。
**为什么收得动**：边际成本极低（毛利接近纯软件），标杆校跑通后靠就业口渠道复制，嵌入就业流程后续费率高。

### 获客层：C 端应届生（免费 + 数据飞轮）

学生免费用求职决策预演——**不指望直接变现**，它是：
1. **流量入口**：口碑传播带来校内渗透，反推学校采购
2. **数据飞轮**：用的学生越多，市场供需模拟越准，产品壁垒越深

轻量增值（单次深度报告等）可做，但不是主营收，不依赖它撑财务。

### 第二曲线：企业反向品牌报告 + 招聘平台集成

- **企业（B）**：出"你公司在应届生眼里的真实画像"报告（hiring_bar 相对位置、culture 印象、同行隐性门槛对比、JD 吸引的候选人画像）。目标客户：年校招 500+ 的中大型科技/快消/金融企业 HR，年度订阅、按需定制。
- **招聘平台（含主办方智联）**：我们的沙盘提升平台投递质量与用户留存，可走集成或分润。**这也是最干净的资本退出故事**——被招聘平台或教育集团（就业服务方向）收购。

### 飞轮闭环

```
学生免费用沙盘 → 数据让市场模拟更准 → 学校拿到更准的群体洞察
       ↑                                          ↓
       └──── 学校采购 + 推广 → 更多学生使用 ──────┘
   （企业/平台洞察作为第二曲线，从同一份市场数据二次变现）
```

## P9 · 数据合规（评委关键关注点）

### 我们所有数据都是合成的

| 数据类型 | 来源 | 合规处理 |
|---|---|---|
| 49 家虚构公司画像 | LLM 生成 + 模板填充 | 代号化（A 厂 / AI-α / 金-2 等），不挂任何真公司名 |
| 200 个求职者 persona | LLM 生成 | 化名（"张同学"等），零真实信息 |
| HR Agent 行为 | 公开 JD 风格 + 公开面试题库泛化 | 不引用任何真实 HR 行为案例 |
| 用户上传材料 | 用户自己提供 | 仅 session 内存使用，**不入数据库**，关闭浏览器即丢 |

### sanitizer 双层兜底

```
Layer 1 - prompt 约束：
  "不准出现任何真实公司名（包括 inspired_by_hint）。
   用'某头部短视频平台量级'这种泛指。"

Layer 2 - 落盘前扫描：
  字典匹配 47 条真公司名（字节/腾讯/阿里/华为/...）+ 顶尖院校名
  发现立即替换为代号
```

实际效果：5/49 家公司在 LLM 生成时漏写了真名，**100% 被 sanitizer 在落盘前替换**。

### 数据合规声明（提交材料独立 1 页）

我们承诺：

1. 用户上传的简历仅 session 内存使用，进程退出即销毁
2. 不收集 PII（个人身份信息）
3. 不向第三方共享任何数据
4. 沙盘里所有 49 公司 + 200 求职者全部为合成数据，零真人参与

## P10 · 市场策略与增长路径

**为什么是现在（Timing）**：1158 万应届生（教育部 2024）涌入市场、就业率/就业质量考核趋严、高校就业服务数字化采购上升——to G 的求职决策与人才洞察工具正当其时。

### Phase 1（0-6 个月）：标杆校跑通 + C 端获客

- C 端：免费 Web + 小程序，校内口碑 + 求职博主传播，积累使用数据
- G 端：拿下 3-5 所强就业导向标杆校（就业中心试点），打磨群体洞察报告
- **目标**：5 所付费标杆校 + 5 万学生使用

### Phase 2（6-12 个月）：就业口渠道复制 + 企业第二曲线

- 用标杆校案例 + 教育口渠道（就业指导协会、校招联盟）规模化获客
- 启动 B 端企业反向品牌报告（年校招 500+ 企业）
- **目标**：50-100 所付费校，ARR 1000-1500 万

### Phase 3（12-24 个月）：平台集成 + 资本化

- 与招聘平台（含智联）谈集成/分润，作为人才洞察数据层
- 退出路径：被招聘平台或教育就业服务集团收购
- **目标**：300+ 校覆盖，进入 A 轮 / 战略并购

### 北极星指标

- 主战场：付费校数 + 续费率（嵌入就业流程后的粘性）
- 获客层：学生使用渗透率（单校内用过的应届生占比）
- 数据飞轮：模拟准确度随数据量提升的曲线

## P11 · 商业可行性（为什么这个模式成立）

> 重模式不重财务精度——这里讲清"为什么能赚钱、钱从哪来、为什么持续"。具体定价待商业化阶段按市场验证。

### 单位经济：高毛利订阅制

- **边际成本极低**：每多服务一所学校，增量成本主要是 LLM 调用，量级很小，毛利结构接近纯软件 SaaS。
- **收入可预测**：高校按年订阅，现金流稳定、可续费，不依赖一次性买断或冲动消费。
- **不靠烧钱**：C 端免费但不大额投放，靠校内口碑自然渗透；钱花在"渠道 BD"而非"流量采买"。

### 为什么这个商业模式能成立

1. **付费方有钱、有动机、好触达**：高校就业中心有预算、有就业率考核 KPI、决策链短（一个主任拍板）——比千万分散的 C 端个体好做得多。
2. **价值可量化**：学校能直接看到"本校群体竞争力分布、技能缺口、该对接哪些雇主"，对就业率 KPI 有实感帮助——付费理由硬。
3. **一份数据、多方变现**：同一套人才市场模拟数据，对学校是群体洞察、对企业是反向品牌、对平台是匹配增强——边际变现成本低。

### 长期壁垒：数据飞轮

用的学校和学生越多 → 市场供需数据越厚 → 模拟越准 → 洞察越值钱 → 更多学校采购。早期标杆校 + 就业口渠道形成先发壁垒。

### 目标客户画像

- **G 端（主战场）**：本科院校就业指导中心，尤其就业率考核压力大、生源去向分散的强就业导向院校（理工 / 财经 / 综合类一本、双一流为先）。
- **B 端（第二曲线）**：年校招 500+ 的中大型科技 / 互联网 / 快消 / 金融企业 HR，需要"年轻人怎么看我"的反向品牌洞察。

## P13 · 风险与缓解

| 风险 | 缓解策略 |
|---|---|
| **数据合规**（评委 #1 关注） | synthetic 数据 + sanitizer 双层 + 用户材料仅会话内存（已实现） |
| **LLM 成本爆炸** | tier 路由（PRIMARY Claude/DeepSeek，群演 Qwen-Turbo），单次 sim < 0.1 元 |
| **sim 真实性质疑** | 复赛/决赛阶段接入公开真实公司数据建模 + 第三方业内人士背书 |
| **用户上传简历隐私** | 仅 session 内存，进程退出即销毁，不入库 |
| **同类竞品快速抄袭** | 工程深度（Multi-Agent + 反事实 + 动态市场）+ 持续迭代 + 早期抢占用户心智 |
| **招聘平台抵触合作** | 我们的价值是"导流 + 提质"，不抢平台核心业务（投递撮合），合作而非竞争 |
| **应届生付费意愿低** | 春招/秋招前后 4 周是付费高峰，其余时间 freemium 留存。决策咨询是高客单补充 |
